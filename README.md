Ce projet est une application web développée dans le cadre de ProjetRestSoa, composée d’un backend REST en JEE/JAX‑RS et d’un frontend HTML5/CSS3/JavaScript ES6 utilisant la Fetch API pour consommer les services REST; l’objectif est de gérer une liste de personnes avec toutes les opérations CRUD, c’est‑à‑dire l’affichage dynamique d’un tableau listant les personnes, l’ajout via un formulaire avec une validation minimale côté client, la modification avec pré‑remplissage des champs et mise à jour, la suppression avec une demande de confirmation, ainsi que la recherche par identifiant et par nom en fonction des endpoints exposés; côté frontend, l’application repose sur plusieurs pages (index.html, add.html, edit.html, search.html) qui manipulent le DOM et envoient des requêtes AJAX vers le backend, tandis que le backend repose sur Java 8, Tomcat 9 (Servlet 4.0), JAX‑RS (implémentation Jersey), JPA/Hibernate et une base de données MySQL 8 configurée avec le driver mysql-connector-java:5.1.48, et expose les principaux endpoints REST GET /persons, GET /persons/{id}, POST /persons, PUT /persons/{id}, DELETE /persons/{id} et GET /persons/search?nom=..., permettant ainsi au frontend de réaliser toutes les opérations de création, lecture, mise à jour et suppression de personnes; une vidéo explicative accompagne ce projet et présente brièvement l’architecture choisie avant de montrer en détail, dans le navigateur, le fonctionnement de chaque fonctionnalité de gestion des personnes, depuis la configuration initiale jusqu’aux différents scénarios d’utilisation.
